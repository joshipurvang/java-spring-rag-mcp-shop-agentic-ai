Semantic match — not keyword match
User said "ML work" — product says "RTX 4070, creative work, OLED"

Reads result, decides next tool
reads @Tool desc: "always call checkStock BEFORE ordering" → calls checkStock

→ Dell XPS 15 OLED · 5 units · IN STOCK · $1799.99

→ Order ORD-1001 confirmed · $1799.99 · email sent to sam@example.com
8

Synthesises friendly response from all tool results
"Great! I've found and ordered the Dell XPS 15 OLED for you. Order ORD-1001 is confirmed at $1799.99. You'll receive a confirmation at sam@example.com."
Zero routing code — the LLM decided steps 3 → 6 → 7 by reading @Tool description text
Why "agentic"?
A normal chatbot answers from training data. This agent autonomously decides which tools to call and loops until it has everything it needs. That think → act → observe cycle is the agent pattern.
RAG role
semanticSearch() doesn't do keyword matching. It embeds the query, searches Redis vectors by cosine similarity, and returns semantically related products even when no exact words match.
@Tool descriptions
The LLM reads the description string on each @Tool method to decide when to call it. Write them precisely. "Always call checkStock before ordering" is why step 6 happened before step 7.
MCP server
The same placeOrder(), checkStock(), and semanticSearch() methods are exposed via /sse. Claude Desktop can call them directly without going through ShopAgent at all.
